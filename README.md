\# Práctica: Configuración de GitHub y Plantilla de Issues

\## Autor: Hugo Moncayo



\## Propósito de la práctica

Este repositorio se utiliza para una práctica de Aseguramiento de Calidad de Software (SQA). El objetivo principal es establecer las bases de un entorno de desarrollo profesional en GitHub, organizando una estructura de directorios estandarizada y configurando plantillas de \*issues\* que permitan un reporte de errores y mejoras con rigor técnico.



\## Objetivos de aprendizaje

\- \*\*Configuración Profesional:\*\* Establecer un flujo de trabajo basado en la rama `main`.

\- \*\*Organización de Proyectos:\*\* Implementar una estructura de carpetas que facilite la trazabilidad del software.

\- \*\*Gestión de Calidad:\*\* Crear plantillas de \*issues\* para estandarizar la comunicación entre QA y Desarrollo.

\- \*\*Control de Versiones:\*\* Aplicar principios fundamentales de Git para la documentación del ciclo de vida del proyecto.



\## Estructura del repositorio

Para garantizar la calidad y el orden, el proyecto sigue esta jerarquía:

\- \*\*.github/ISSUE\_TEMPLATE\*\*: Contiene las plantillas de reportes (Bugs y Mejoras).

\- \*\*docs\*\*: Repositorio de la documentación general, requerimientos y planes de prueba.

\- \*\*src\*\*: Directorio destinado al código fuente del sistema.

\- \*\*tests\*\*: Carpeta crítica para el QA, donde se almacenan pruebas manuales, automatizadas y resultados de ejecución.



\## Herramientas utilizadas

\- \*\*Git\*\*: Sistema de control de versiones.

\- \*\*GitHub\*\*: Plataforma de alojamiento y gestión de proyectos.

\- \*\*Markdown\*\*: Lenguaje de marcado para la documentación técnica.
